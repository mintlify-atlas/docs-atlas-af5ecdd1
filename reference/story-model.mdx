## Overview

The `Story` data class is the core domain model representing a bilingual indigenous legend in Casa de Historias. Each story contains content in both Spanish and Nahuatl, along with rich metadata about its origin, narrator, and location.

## Data Class Definition

```kotlin domain/model/Story.kt
package com.example.casadehistorias.domain.model

data class Story(
    val id: String,
    val titleEs: String,
    val titleNahuatl: String,
    val contentEs: String,
    val contentNahuatl: String,
    val audioUrl: String?,
    val imageUrl: String?,
    val narratorName: String,
    val community: String,
    val latitude: Double? = null,
    val longitude: Double? = null,
    val authorId: String = "",
    val tags: List<String> = emptyList(),
    val createdAt: Long = 0L
)
```

## Field Reference

<ParamField path="id" type="String" required>
  Unique identifier for the story. Generated by Firebase Firestore when the story is created.
</ParamField>

<ParamField path="titleEs" type="String" required>
  Story title in Spanish (español). This is the primary display title in Spanish-speaking contexts.
</ParamField>

<ParamField path="titleNahuatl" type="String" required>
  Story title in Nahuatl. Preserves the indigenous language representation of the legend's name.
</ParamField>

<ParamField path="contentEs" type="String" required>
  Full story content in Spanish. Contains the complete narrative text of the legend.
</ParamField>

<ParamField path="contentNahuatl" type="String" required>
  Full story content in Nahuatl. Contains the indigenous language version of the legend.
</ParamField>

<ParamField path="audioUrl" type="String?" default="null">
  URL to the audio narration file stored in Firebase Storage. Optional field for stories with audio recordings.
</ParamField>

<ParamField path="imageUrl" type="String?" default="null">
  URL to the cover image stored in Firebase Storage. Optional field for stories with visual accompaniment.
</ParamField>

<ParamField path="narratorName" type="String" required>
  Name of the person who narrated or contributed this legend. Essential for attribution and cultural preservation.
</ParamField>

<ParamField path="community" type="String" required>
  Name of the indigenous community from Sierra de Zongolica where this legend originates. Used for filtering and geographic organization.
</ParamField>

<ParamField path="latitude" type="Double?" default="null">
  Geographic latitude coordinate of the story's origin location. Used for map visualization features.
</ParamField>

<ParamField path="longitude" type="Double?" default="null">
  Geographic longitude coordinate of the story's origin location. Used for map visualization features.
</ParamField>

<ParamField path="authorId" type="String" default='""'>
  Firebase Auth user ID of the person who submitted this story to the app. Links stories to user profiles.
</ParamField>

<ParamField path="tags" type="List<String>" default="emptyList()">
  Searchable tags for categorizing and discovering stories. Examples: "mythology", "nature", "ancestral".
</ParamField>

<ParamField path="createdAt" type="Long" default="0L">
  Unix timestamp (milliseconds) when the story was created. Used for sorting and displaying creation dates.
</ParamField>

## Usage Examples

### Creating a New Story

```kotlin
val story = Story(
    id = "abc123",
    titleEs = "La Leyenda del Cerro Sagrado",
    titleNahuatl = "Tepētl Tēōcalli",
    contentEs = "Hace muchos años, en las montañas...",
    contentNahuatl = "Miac xihuitl, itech tepemeh...",
    audioUrl = "https://storage.googleapis.com/stories/abc123/audio.mp3",
    imageUrl = "https://storage.googleapis.com/stories/abc123/cover.jpg",
    narratorName = "María González",
    community = "Tequila",
    latitude = 18.6586,
    longitude = -97.0031,
    authorId = "user_xyz789",
    tags = listOf("montaña", "sacred", "ancestros"),
    createdAt = System.currentTimeMillis()
)
```

### Displaying Bilingual Content

```kotlin
@Composable
fun StoryHeader(story: Story, language: String) {
    val title = when (language) {
        "nahuatl" -> story.titleNahuatl
        else -> story.titleEs
    }
    
    Text(
        text = title,
        style = MaterialTheme.typography.headlineMedium
    )
}
```

## Related Models

<Info>
  The `Story` model has two related representations:
  - **StoryFirestore**: Firebase-specific model with `Timestamp` fields for Firestore serialization
  - **StoryEntity**: Room database entity for offline caching with local file paths
  
  See [Firebase Integration](/reference/firebase-integration) and [Room Database](/reference/room-database) for details.
</Info>

## Best Practices

<Warning>
  Always validate that both `titleEs` and `titleNahuatl` are non-empty before creating a story. The app's mission is to preserve both languages equally.
</Warning>

- Use descriptive, searchable tags to improve story discoverability
- Include geographic coordinates when available to enable map-based exploration
- Store audio and image files in Firebase Storage before setting URLs
- Preserve narrator attribution for cultural respect and community recognition
