Casa de Historias provides a comprehensive system for managing indigenous legends from the Sierra de Zongolica region. Community members can contribute, edit, and preserve their cultural stories through an intuitive interface.

## Story Data Model

Each story in the app is represented by the `Story` data class, which captures both Spanish and Nahuatl content:

```kotlin Story.kt:3
data class Story(
    val id: String,
    val titleEs: String,
    val titleNahuatl: String,
    val contentEs: String,
    val contentNahuatl: String,
    val audioUrl: String?,
    val imageUrl: String?,
    val narratorName: String,
    val community: String,
    val latitude: Double? = null,
    val longitude: Double? = null,
    val authorId: String = "",
    val tags: List<String> = emptyList(),
    val createdAt: Long = 0L
)
```

<Info>
The Story model is designed to preserve both languages equally, ensuring that indigenous knowledge is maintained in its original form while remaining accessible to Spanish speakers.
</Info>

The `StoryRepository` provides multiple methods for retrieving stories based on different criteria:

<Steps>


### Publishing Stories

The repository's `addStory` method handles story creation with authentication:

```kotlin StoryRepository.kt:52
suspend fun addStory(story: Story): String? {
    val currentUser = authRepository.currentUser
    return if (currentUser != null) {
        val storyFirestore = story.toFirestoreModel(currentUser.uid)
        firestoreRepository.addStory(storyFirestore)
    } else null
}
```

## Repository Pattern

The app uses a repository pattern to abstract data access:

<CardGroup cols={2}>
  <Card title="StoryRepository" icon="database">
    Provides a clean API for story operations and manages data transformation between Firestore and domain models.
  </Card>
  <Card title="ViewModel Layer" icon="layer-group">
    StoryViewModel manages UI state and coordinates repository calls with lifecycle-aware data streams.
  </Card>
</CardGroup>

```kotlin StoryViewModel.kt:37
private fun loadStories() {
    viewModelScope.launch {
        _isLoading.value = true
        _errorMessage.value = null
        repository.getAllStoriesFlow()
            .catch { e ->
                Log.e("StoryViewModel", "Error cargando historias: ${e.message}")
                _errorMessage.value = "Error al cargar historias: ${e.message}"
                _isLoading.value = false
            }
            .collect { storyList ->
                _stories.value = storyList
                _isLoading.value = false
            }
    }
}
```

<Tip>
The repository pattern allows for easy testing and future data source changes (e.g., adding offline caching) without modifying UI code.
</Tip>
