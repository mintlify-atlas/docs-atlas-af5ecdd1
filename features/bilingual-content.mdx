Casa de Historias is fundamentally designed as a bilingual application, preserving indigenous legends in both Spanish and Nahuatl. This dual-language approach ensures that cultural knowledge is maintained in its original form while remaining accessible to broader audiences.

## The Story Data Model

Every story in the system contains parallel fields for both languages:

```kotlin Story.kt:3
data class Story(
    val id: String,
    val titleEs: String,           // Spanish title
    val titleNahuatl: String,       // Nahuatl title
    val contentEs: String,          // Spanish content
    val contentNahuatl: String,     // Nahuatl content
    val audioUrl: String?,
    val imageUrl: String?,
    val narratorName: String,
    val community: String,
    val latitude: Double? = null,
    val longitude: Double? = null,
    val authorId: String = "",
    val tags: List<String> = emptyList(),
    val createdAt: Long = 0L
)
```

<Info>
The bilingual structure is built directly into the data model, treating both languages as first-class citizens rather than relegating one to a translation layer.
</Info>

## Cultural Importance

The bilingual approach serves multiple critical purposes:

<CardGroup cols={2}>
  <Card title="Language Preservation" icon="landmark">
    Nahuatl is an endangered indigenous language. By storing stories in their original language, the app helps preserve linguistic heritage for future generations.
  </Card>
  <Card title="Cultural Authenticity" icon="certificate">
    Stories told in Nahuatl retain nuances, idioms, and cultural concepts that may be lost in translation. The original language preserves the narrator's authentic voice.
  </Card>
  <Card title="Educational Value" icon="graduation-cap">
    Bilingual presentation allows Nahuatl learners to study the language in context, comparing translations and understanding how concepts are expressed.
  </Card>
  <Card title="Community Accessibility" icon="users">
    Spanish translations ensure that younger generations who may not speak Nahuatl fluently can still connect with their cultural heritage.
  </Card>
</CardGroup>

## Optional Nahuatl Content

While the app strongly encourages bilingual content, it recognizes practical challenges:

<Warning>
Not all community members are fluent in Nahuatl. The app allows Spanish-only submissions to avoid creating barriers to participation, while still providing the infrastructure for full bilingual preservation.
</Warning>

### Validation Logic

```kotlin AddStoryScreen.kt:280
titleError = titleEs.isBlank()      // Spanish required
contentError = contentEs.isBlank()  // Spanish required
narratorError = narratorName.isBlank()
// Note: titleNahuatl and contentNahuatl are NOT validated

if (!titleError && !contentError && !narratorError) {
    viewModel.addStory(
        titleEs = titleEs,
        titleNahuatl = titleNahuatl,        // Can be empty
        contentEs = contentEs,
```

## Best Practices

<Steps>
  <Step title="Encourage Complete Bilingual Entries">
    Community coordinators should work with elders and Nahuatl speakers to provide translations for Spanish-only submissions.
  </Step>

  <Step title="Preserve Original Language">
    When a story is originally told in Nahuatl, that version should be the primary source, with Spanish as the translation.
  </Step>

  <Step title="Document Dialect Variations">
    The Nahuatl language has regional variations. Use the community field and tags to note specific dialect information.
  </Step>

  <Step title="Maintain Parallel Quality">
    Both language versions should be equally well-written and complete, not just literal word-for-word translations.
  </Step>
</Steps>

## Future Language Support

The bilingual architecture is extensible. If needed, additional indigenous languages from the Sierra de Zongolica region (such as Totonac or other Nahua variants) could be added by:

1. Expanding the Story data model with new language fields
2. Adding corresponding UI fields in the entry forms
3. Updating the display logic to show additional language sections

<Info>
The current Spanish-Nahuatl focus reflects the primary languages of the target communities, but the system is designed to accommodate linguistic diversity.
</Info>
