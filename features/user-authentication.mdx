Casa de Historias uses Firebase Authentication to manage user accounts, ensuring that community members can contribute stories while maintaining accountability and security.

## Authentication Architecture

The app implements a repository pattern for authentication:

```kotlin AuthRepository.kt:16
@Singleton
class AuthRepository @Inject constructor(
    private val auth: FirebaseAuth,
    private val firestore: FirebaseFirestore
) {
    val currentUser: FirebaseUser? get() = auth.currentUser

    val authState: Flow<FirebaseUser?> = callbackFlow {
        val listener = FirebaseAuth.AuthStateListener { auth ->
            trySend(auth.currentUser)
        }
        auth.addAuthStateListener(listener)
        awaitClose { auth.removeAuthStateListener(listener) }
    }
}
```

<Info>
The repository exposes both a synchronous `currentUser` property for immediate access and a reactive `authState` Flow for observing authentication changes throughout the app lifecycle.
</Info>

## Sign Up Flow

New users register with email, password, display name, and optional community information:

### Sign Up Method

```kotlin AuthRepository.kt:41
suspend fun createUserWithEmailAndPassword(
    email: String, 
    password: String,
    displayName: String,
    community: String
): Result<FirebaseUser?> {
    return try {
        val result = auth.createUserWithEmailAndPassword(email, password).await()
        result.user?.let { user ->
            createUserProfile(user.uid, email, displayName, community)
        }
        Result.success(result.user)
    } catch (e: Exception) {
        Result.failure(e)
    }
}
```

### User Profile Creation

When a new account is created, a corresponding user profile document is created in Firestore:

```kotlin AuthRepository.kt:83
private suspend fun createUserProfile(
    userId: String, 
    email: String, 
    displayName: String,
    community: String
) {
    val profile = UserProfile(
        id = userId,
        email = email,
        displayName = displayName,
        community = community,
        createdAt = Timestamp.now(),
        lastLogin = Timestamp.now()
    )
    firestore.collection("users").document(userId).set(profile).await()
}
```

<Steps>
  <Step title="Firebase Authentication">
    User account is created in Firebase Auth with email and password
  </Step>

  <Step title="User Profile Document">
    Additional user data (name, community) is stored in Firestore under `users/{userId}`
  </Step>

  <Step title="Timestamp Recording">
    Creation time and last login are recorded for analytics and user management
  </Step>
</Steps>

### Validation Logic

```kotlin SignUpScreen.kt:52
fun validate(): Boolean {
    var valid = true
    emailError = null; passwordError = null; confirmError = null; nameError = null

    if (displayName.isBlank()) {
        nameError = "El nombre es obligatorio"
        valid = false
    }
    if (email.isBlank()) {
        emailError = "El correo es obligatorio"
        valid = false
    } else if (!android.util.Patterns.EMAIL_ADDRESS.matcher(email).matches()) {
        emailError = "Ingresa un correo válido"
        valid = false
    }
    if (password.length < 6) {
        passwordError = "Mínimo 6 caracteres"
        valid = false
    }
    if (password != confirmPassword) {
        confirmError = "Las contraseñas no coinciden"
        valid = false
    }
    return valid
}
```

## Sign In Flow

Existing users sign in with email and password:

```kotlin AuthRepository.kt:31
suspend fun signInWithEmailAndPassword(email: String, password: String): Result<FirebaseUser?> {
    return try {
        val result = auth.signInWithEmailAndPassword(email, password).await()
        updateLastLogin(result.user?.uid)
        Result.success(result.user)
    } catch (e: Exception) {
        Result.failure(e)
    }
}
```

### Last Login Tracking

```kotlin AuthRepository.kt:100
private suspend fun updateLastLogin(userId: String?) {
    userId?.let {
        firestore.collection("users")
            .document(it)
            .update("lastLogin", Timestamp.now())
            .await()
    }
}
```

<Tip>
Tracking last login helps identify active users and can be used for engagement analytics or to prompt inactive users to contribute new stories.
</Tip>

## User Profile Management

Retrieve and update user profiles:

```kotlin AuthRepository.kt:62
suspend fun getUserProfile(userId: String): UserProfile? {
    return try {
        val document = firestore.collection("users").document(userId).get().await()
        document.toObject(UserProfile::class.java)
    } catch (e: Exception) {
        null
    }
}

suspend fun updateUserProfile(userId: String, profile: UserProfile): Boolean {
    return try {
        firestore.collection("users")
            .document(userId)
            .set(profile.copy(id = userId))
            .await()
        true
    } catch (e: Exception) {
        false
    }
}
```

## Security Considerations

<Steps>
  <Step title="Password Requirements">
    Minimum 6 characters enforced (Firebase default). Consider increasing to 8+ for production.
  </Step>

  <Step title="Email Verification">
    Currently not implemented but recommended for production to prevent fake accounts.
  </Step>

  <Step title="Firestore Security Rules">
    Ensure Firestore rules restrict story editing to the original author:
    ```
    match /stories/{storyId} {
      allow update, delete: if request.auth.uid == resource.data.authorId;
    }
    ```
  </Step>

  <Step title="Rate Limiting">
    Consider implementing rate limits on story creation to prevent spam.
  </Step>
</Steps>
